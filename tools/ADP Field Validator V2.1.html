User Guide & Best Practices

How to Use the Validator

1 Load Data

Upload a data file or paste rows directly into the textarea. Ensure your Delimiter matches the file (Pipe is default).

2 Inspect Results

Rows with field count mismatches or empty required fields turn red. Use filters to isolate workers.

3 Investigation

Hover headers to see Mapping Logic and Workday Source to confirm data integrity against technical requirements.

Using the Schema Editor

The Schema Editor allows you to customize the validation rules for any pipe-delimited interface.

  • Modify Metadata: Edit "Field Label", "Mapping Logic", and "Workday Source" directly in the table. These values appear in the tooltips on the Validator tab.
  • Validation Rules: Check the Mandatory box to highlight empty fields in red. Check the Optional box to shade the field grey (OK to be null).
  • Manage Records: Use the + New Type button to add headers or subtypes (e.g., 05, 09). Use Delete Current to remove unused schema definitions.
  • Structure Adjustment: Use + Add New Field Position to append a column to the end of the record type. Use the × button to remove a specific position.
  • Persistence: Click Apply & Save Schema to update the local session logic.

AI Acceleration workflow

To quickly set up a new interface schema without manual entry, follow this workflow:

  1. Define your basic Record Types (e.g., 01, 02) in the Schema Editor.
  2. Use the Export Setup button to save the configuration as a JSON file.
  3. Upload this JSON file to an AI tool (like ChatGPT or Gemini) along with the text from your Technical Design (TD) or BRD tables.
  4. Ask the AI: "Update the 'logic' and 'source' values in this JSON based on the provided technical documentation for each field position."
  5. Re-import the updated JSON file back into this tool via the Import Setup button.

SECURITY REQUIREMENT

Before uploading ANY configuration data to external AI tools, you MUST explicitly remove all PII (Personally Identifiable Information), copyrighted content, or company intellectual property. Only share generic field structures and technical mapping descriptions.
